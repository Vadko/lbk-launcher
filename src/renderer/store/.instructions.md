# Zustand Store Instructions

This directory contains Zustand state stores for the React application.

## Structure

```
src/renderer/store/
├── useStore.ts             # Main application state
├── useSettingsStore.ts     # User preferences (persisted)
├── useModalStore.ts        # Modal visibility state
├── useConfirmStore.ts      # Confirmation dialog state
├── useGamepadModeStore.ts  # Steam Deck gamepad navigation
├── usePromoModalStore.ts   # Promotional modal state (persisted)
├── useSubscriptionsStore.ts # Game/team subscriptions (persisted)
├── electronStorage.ts      # Electron-compatible storage adapter
└── subscriptions/          # Subscription-related utilities
```

## Store Patterns

### Basic Store

```typescript
import { create } from 'zustand';

interface ExampleStore {
  value: string;
  setValue: (value: string) => void;
}

export const useExampleStore = create<ExampleStore>((set) => ({
  value: '',
  setValue: (value) => set({ value }),
}));
```

### Persisted Store

```typescript
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';
import { electronStorage } from './electronStorage';

export const useSettingsStore = create<SettingsStore>()(
  persist(
    (set, get) => ({
      // State and actions
    }),
    {
      name: 'settings-storage',
      storage: createJSONStorage(() => electronStorage),
      partialize: (state) => ({
        // Only persist these fields
        sortOrder: state.sortOrder,
        animationsEnabled: state.animationsEnabled,
      }),
    }
  )
);
```

## Usage in Components

### Selecting State

```typescript
// GOOD - Select specific slices
const selectedGame = useStore((s) => s.selectedGame);
const setSelectedGame = useStore((s) => s.setSelectedGame);

// BAD - Causes re-render on ANY state change
const store = useStore();
```

### Multiple Selectors

```typescript
// Use shallow comparison for multiple values
import { shallow } from 'zustand/shallow';

const { selectedGame, searchQuery } = useStore(
  (s) => ({ selectedGame: s.selectedGame, searchQuery: s.searchQuery }),
  shallow
);
```

### Actions Outside Components

```typescript
// Access store without hook
useStore.getState().setSelectedGame(game);
useSettingsStore.getState().toggleAnimations();
```

## Persistence

### Electron Storage Adapter

Uses `electron-store` via IPC for persistence:

```typescript
// electronStorage.ts
export const electronStorage: StateStorage = {
  getItem: async (name) => {
    return await window.electronAPI.storeGet(name);
  },
  setItem: async (name, value) => {
    await window.electronAPI.storeSet(name, value);
  },
  removeItem: async (name) => {
    await window.electronAPI.storeDelete(name);
  },
};
```

### Partial Persistence

Only persist necessary fields:

```typescript
persist(
  (set) => ({ /* ... */ }),
  {
    name: 'storage-key',
    partialize: (state) => ({
      // Only these fields are saved
      favoriteGameIds: state.favoriteGameIds,
      sortOrder: state.sortOrder,
    }),
  }
)
```

## Guidelines

1. **Keep stores focused** - One store per domain (settings, modal, etc.)
2. **Select specific slices** - Avoid re-renders from unrelated changes
3. **Use actions** - Don't set state directly from components
4. **Persist sparingly** - Only user preferences, not transient UI state
5. **Type everything** - Full TypeScript interfaces for stores
