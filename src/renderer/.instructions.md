# Renderer Process Instructions

This directory contains the React UI running in Electron's renderer process (browser environment).

## Environment

- **Runtime**: Chromium browser (no Node.js APIs)
- **Framework**: React 19 with TypeScript
- **State**: Zustand stores
- **Styling**: TailwindCSS + framer-motion
- **IPC**: Via `window.electronAPI` (exposed by preload)

## Key Patterns

### Component Structure

Use feature folders with named exports:

```typescript
// src/renderer/components/GameCard/GameCard.tsx
export const GameCard: React.FC<GameCardProps> = ({ game }) => {
  // Component implementation
};

// src/renderer/components/GameCard/index.ts
export { GameCard } from './GameCard';
export type { GameCardProps } from './GameCard';
```

### State Management (Zustand)

```typescript
// Access store
import { useStore } from '@store/useStore';

const Component = () => {
  // Select specific state slices for performance
  const selectedGame = useStore((s) => s.selectedGame);
  const setSelectedGame = useStore((s) => s.setSelectedGame);
  
  // AVOID: Selecting entire store
  // const store = useStore(); // Re-renders on ANY state change
};
```

### IPC Communication

Always use `window.electronAPI`:

```typescript
// CORRECT - Use exposed API
const games = await window.electronAPI.fetchGames(params);

// WRONG - No direct Node.js access
import fs from 'fs'; // ERROR: fs not available in renderer
```

### Event Listeners

Use effect cleanup:

```typescript
useEffect(() => {
  const unsubscribe = window.electronAPI.onDownloadProgress((gameId, progress) => {
    // Handle progress
  });
  
  return () => unsubscribe();
}, []);
```

## Styling Guidelines

### TailwindCSS

Use Tailwind utilities, avoid inline styles:

```tsx
// CORRECT
<div className="flex items-center gap-2 p-4 bg-gray-800 rounded-lg">

// AVOID
<div style={{ display: 'flex', alignItems: 'center' }}>
```

### Animations

Respect user preference for reduced motion:

```tsx
import { useSettingsStore } from '@store/useSettingsStore';
import { motion } from 'framer-motion';

const Component = () => {
  const { animationsEnabled } = useSettingsStore();
  
  return (
    <motion.div
      initial={animationsEnabled ? { opacity: 0 } : false}
      animate={{ opacity: 1 }}
    />
  );
};
```

### Dark Theme

App uses dark theme exclusively. Use gray-800/900 backgrounds, cyan accents:

```tsx
<button className="bg-cyan-600 hover:bg-cyan-500 text-white">
```

## Component Best Practices

### Props Typing

```typescript
interface GameCardProps {
  game: Game;
  onSelect?: (game: Game) => void;
  isSelected?: boolean;
}

export const GameCard: React.FC<GameCardProps> = ({
  game,
  onSelect,
  isSelected = false,
}) => {
  // ...
};
```

### Error Boundaries

Wrap feature areas with error boundaries:

```tsx
<ErrorBoundary fallback={<ErrorFallback />}>
  <FeatureComponent />
</ErrorBoundary>
```

### Loading States

Use skeleton loaders for async content:

```tsx
const { data, isLoading } = useQuery(...);

if (isLoading) {
  return <GameCardSkeleton />;
}
```

## Stores Reference

| Store | Purpose | Persisted |
|-------|---------|-----------|
| `useStore` | Main app state, games, installation | Partial |
| `useSettingsStore` | User preferences | Yes |
| `useModalStore` | Modal visibility | No |
| `useGamepadModeStore` | Steam Deck navigation | No |
| `useSubscriptionsStore` | Game/team subscriptions | Yes |
| `useConfirmStore` | Confirmation dialogs | No |
| `usePromoModalStore` | Promotional modals | Yes |

## Performance Guidelines

1. **Memoize callbacks** with `useCallback` for event handlers passed to children
2. **Select minimal state** from Zustand stores
3. **Virtualize long lists** - Don't render 1000 game cards at once
4. **Lazy load heavy components** with `React.lazy`
5. **Debounce search input** - Don't query on every keystroke

```tsx
const { isGamepadMode, navigationArea } = useGamepadModeStore();

// Adjust UI for gamepad (larger hit targets, focus indicators)
<button className={isGamepadMode ? 'p-4' : 'p-2'}>
```
