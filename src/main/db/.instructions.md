# Database Module Instructions

This module handles local SQLite database and Supabase synchronization.

## Module Structure

```
src/main/db/
├── database.ts           # SQLite database manager (singleton)
├── migrations.ts         # Database schema migrations
├── db-queries.ts         # Direct SQL query functions
├── db-worker.ts          # Worker thread for heavy operations
├── db-worker-client.ts   # Main thread client for worker
├── games-repository.ts   # Game data repository pattern
├── banners-api.ts        # Ad banners management
├── supabase-credentials.ts # Supabase connection management
├── supabase-realtime.ts  # Real-time subscriptions
├── supabase-sync-api.ts  # Sync operations
└── sync-manager.ts       # Sync orchestration
```

## Database Schema

```sql
-- Main games table
CREATE TABLE games (
  id TEXT PRIMARY KEY,
  slug TEXT UNIQUE,
  name TEXT NOT NULL,
  team TEXT,
  description TEXT,
  cover_url TEXT,
  downloads INTEGER DEFAULT 0,
  version TEXT,
  install_paths JSONB,        -- Array of InstallPath
  platforms TEXT[],           -- Supported OS platforms
  created_at TIMESTAMP,
  updated_at TIMESTAMP,
  ...
);

-- Full-text search
CREATE VIRTUAL TABLE games_fts USING fts5(
  name, team, description,
  content=games
);

-- Spellfix for typo-tolerant search
CREATE VIRTUAL TABLE games_spellfix USING spellfix1;
```

## Key Classes

### DatabaseManager (Singleton)

```typescript
const db = DatabaseManager.getInstance();

// Direct queries (use sparingly)
const result = db.prepare('SELECT * FROM games WHERE id = ?').get(id);
```

### GamesRepository

```typescript
const repo = GamesRepository.getInstance();

// Common operations
const game = repo.getGameById(id);
repo.upsertGame(game);
repo.upsertGames(games);  // Batch upsert
repo.incrementDownloads(gameId);
repo.deleteGame(gameId);

// Search with filtering
const results = repo.getGames({
  searchQuery: 'witcher',
  hideAiTranslations: true,
  sortOrder: 'downloads',
  statuses: ['completed'],
});
```

### Worker Thread

Heavy operations run in worker to avoid blocking main thread:

```typescript
import { dbWorkerClient } from './db-worker-client';

// Initialize worker on app start
await dbWorkerClient.init();

// Batch upsert in worker thread
await dbWorkerClient.upsertGames(games);
```

## Migrations

**NEVER modify existing migrations. Always add new ones.**

```typescript
// migrations.ts
const migrations = [
  {
    version: 1,
    up: (db) => {
      db.exec(`CREATE TABLE games (...)`);
    },
  },
  {
    version: 2,
    up: (db) => {
      db.exec(`ALTER TABLE games ADD COLUMN new_field TEXT`);
    },
  },
];
```

Migration runs automatically on app start in `database.ts`.

## Supabase Sync

### Initial Sync

```typescript
import { SyncManager } from './sync-manager';
import { fetchAllGamesFromSupabase } from './supabase-sync-api';

const syncManager = new SyncManager();
await syncManager.fullSync(fetchAllGamesFromSupabase);
```

### Real-time Updates

```typescript
import { SupabaseRealtimeManager } from './supabase-realtime';

const realtimeManager = new SupabaseRealtimeManager();
realtimeManager.subscribe(
  (game) => { /* handle update */ },
  (gameId) => { /* handle delete */ }
);
```

### Sync Flow

```
App Start
    │
    ├─► Check last sync time
    │
    ├─► If stale (>5min), full sync
    │   └─► Fetch all games from Supabase
    │       └─► Upsert into local SQLite
    │
    └─► Subscribe to real-time changes
        └─► On change, update local DB
            └─► Notify renderer via IPC
```

## Spellfix (Fuzzy Search)

The app uses SQLite's spellfix1 extension for typo-tolerant search:

```typescript
// FTS5 search with variations handles typos
repo.getGames({ searchQuery: 'vitcher' });  // Finds "Witcher"
repo.getGames({ searchQuery: 'ведьмак' });  // Finds "Відьмак"
```

Extension loaded from `resources/extensions/spellfix.{dll,so,dylib}`

## Common Patterns

### Safe Query Execution

```typescript
try {
  const stmt = db.prepare('SELECT * FROM games WHERE id = ?');
  const game = stmt.get(gameId);
  return game;
} catch (error) {
  console.error('[Database] Query failed:', error);
  return null;
}
```

### Transactions

```typescript
const transaction = db.transaction((games: Game[]) => {
  const stmt = db.prepare('INSERT OR REPLACE INTO games ...');
  for (const game of games) {
    stmt.run(game);
  }
});

transaction(gamesToInsert);
```

### JSON Fields

```typescript
// install_paths is stored as JSON string
const game = repo.getGameById(id);
const paths = JSON.parse(game.install_paths);
```

## Performance Guidelines

1. **Use worker for heavy queries** - Full table scans, complex searches
2. **Prepare statements** - Reuse for repeated queries
3. **Use transactions** - For batch inserts/updates
4. **Index frequently queried columns** - id, slug, team
5. **Limit result sets** - Use LIMIT clause

## Clear Cache

Database can be cleared via:
1. Settings UI → Clear Cache
2. Delete marker file triggers cleanup on next start

```typescript
// Creates marker for deletion on next app start
writeFileSync(path.join(userDataPath, 'lbk.db.delete'), '');
```
