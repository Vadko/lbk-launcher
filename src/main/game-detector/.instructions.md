# Game Detector Module Instructions

This module detects installed games across multiple platforms and launchers.

## Module Structure

```
src/main/game-detector/
├── index.ts        # Main entry, re-exports, detection orchestration
├── types.ts        # Shared type definitions
├── steam.ts        # Steam game detection
├── gog.ts          # GOG Galaxy detection
├── epic.ts         # Epic Games Store detection
├── xbox.ts         # Xbox/Microsoft Store detection
├── rockstar.ts     # Rockstar Games Launcher detection
├── heroic.ts       # Heroic Launcher (Linux)
├── lutris.ts       # Lutris (Linux)
├── kurin.ts        # Kurin (Ukrainian mod manager)
└── game-titles.ts  # Game title normalization
```

## Platform-Specific Logic

### Steam Detection

Steam games are located via:
1. Parse `libraryfolders.vdf` for library paths
2. Search each library for `steamapps/common/{gamePath}`

```typescript
// Get Steam installation path
const steamPath = getSteamPath();
// Windows: Registry or common paths
// macOS: ~/Library/Application Support/Steam
// Linux: ~/.steam/steam or ~/.local/share/Steam

// Get all installed games
const games = await getAllInstalledSteamGames();
// Returns Map<installDir, fullPath>

// Get Steam library app IDs
const appIds = getSteamLibraryAppIds();
```

### GOG Detection

GOG games detected via:
1. GOG Galaxy database: `%ProgramData%\GOG.com\Galaxy\storage\galaxy-2.0.db`
2. Registry entries (Windows)

```typescript
const gogPath = findGOGGame('Witcher 3 Wild Hunt GOTY');
const gogId = getGOGGameId('Witcher 3 Wild Hunt GOTY');
```

### Epic Detection

Epic games detected via:
1. Manifest files: `%ProgramData%\Epic\EpicGamesLauncher\Data\Manifests\*.item`

```typescript
const epicPath = findEpicGame('Fortnite');
const appName = getEpicAppName('Fortnite');
```

### Xbox/MS Store Detection

Xbox games use different path structure:
1. `%ProgramFiles%\WindowsApps\` (read permissions required)
2. `%LocalAppData%\Packages\` for UWP app data

```typescript
const xboxPaths = getInstalledXboxGamePaths();
// Returns array of folder names, not full paths
```

### Heroic Launcher (Linux)

Heroic is an open-source launcher for GOG and Epic on Linux:

```typescript
// Epic games via Heroic
const epicAppName = getHeroicEpicAppName('Hades');

// GOG games via Heroic
const gogId = getHeroicGOGId('Witcher 3');
```

## Caching

Game detection results are cached to avoid repeated filesystem scans:

```typescript
// Invalidate Steam cache when library changes
invalidateSteamGamesCache();
invalidateSteamLibraryAppIdsCache();
invalidateSteamPathCache();
```

Cache invalidation triggers:
- `steam-watcher.ts` monitors Steam library changes
- Manual refresh in UI

## Path Matching

Game paths in database may not match exact folder names. Use fuzzy matching:

```typescript
// Database path: "The Witcher 3 Wild Hunt"
// Actual folder: "The Witcher 3 Wild Hunt - Game of the Year Edition"

// game-titles.ts handles normalization
const normalizedTitle = normalizeGameTitle(folderName);
```

## Common Patterns

### Safe Path Checking

```typescript
import { existsSync } from 'fs';

function findGame(basePaths: string[], gamePath: string): string | null {
  for (const base of basePaths) {
    const fullPath = path.join(base, gamePath);
    if (existsSync(fullPath)) {
      return fullPath;
    }
  }
  return null;
}
```

### Registry Reading (Windows)

```typescript
import { execSync } from 'child_process';

function getRegistryValue(key: string, valueName: string): string | null {
  try {
    const output = execSync(
      `reg query "${key}" /v ${valueName}`,
      { encoding: 'utf8' }
    );
    const match = output.match(/REG_SZ\s+(.+)/);
    return match ? match[1].trim() : null;
  } catch {
    return null;
  }
}
```

### VDF Parsing

Steam uses Valve Data Format for config files:

```typescript
import { parse } from 'fast-vdf';

const vdfContent = fs.readFileSync(libraryFoldersPath, 'utf8');
const parsed = parse(vdfContent);
```

## Performance Considerations

1. **Cache Steam library** - VDF parsing is slow
2. **Lazy load platform detectors** - Don't scan all platforms upfront
3. **Use existsSync sparingly** - Batch path checks when possible
4. **Monitor library changes** - Don't rescan on every render
