# Installer Module Instructions

This module handles downloading, extracting, and installing game localizations.

## Module Structure

```
src/main/installer/
├── archive.ts           # Archive extraction (7z, zip, rar)
├── backup.ts            # File backup and restore
├── cache.ts             # Installation metadata cache
├── disk.ts              # Disk space checking
├── download.ts          # File download with progress
├── download-and-extract.ts # Combined download + extraction
├── error-handler.ts     # Error processing and categorization
├── errors.ts            # Custom error classes
├── files.ts             # File system utilities
├── hash.ts              # File hashing (SHA256)
├── platform.ts          # Platform-specific installation
├── proton.ts            # Linux Proton/Wine integration
├── resume.ts            # Download resume functionality
└── uninstall.ts         # Translation removal
```

## Installation Flow

```
1. installTranslation() [installer.ts]
   │
   ├─► checkPlatformCompatibility() [platform.ts]
   │
   ├─► detectGamePath() [game-detector/]
   │
   ├─► checkDiskSpace() [disk.ts]
   │
   ├─► downloadAndExtractArchive() [download-and-extract.ts]
   │   ├─► downloadFile() [download.ts]
   │   └─► extractArchive() [archive.ts]
   │
   ├─► backupFiles() [backup.ts] (if enabled)
   │
   ├─► runInstaller() [platform.ts] (if executable installer)
   │   └─► runProton() [proton.ts] (on Linux)
   │
   └─► saveInstallationInfo() [cache.ts]
```

## Error Classes

Use semantic errors for UI decisions:

```typescript
// Game path couldn't be auto-detected
throw new ManualSelectionError('Game not found');

// Network issues (retry possible)
throw new NetworkError('Download failed');

// API rate limiting
throw new RateLimitError('Too many requests');

// User paused download (not an error)
throw new PausedSignal();
```

## Platform Handling

### Windows

```typescript
// .bat/.cmd files need shell mode
if (file.endsWith('.bat') || file.endsWith('.cmd')) {
  spawn(file, [], { shell: true, cwd: workingDir });
}

// .exe files run directly
spawn(file, args, { cwd: workingDir });
```

### Linux (Proton)

```typescript
// All Windows executables must use Proton
if (isLinux() && isWindowsFile(file)) {
  await runProton(protonPath, file, gamePath, onStatus);
}
```

### macOS

```typescript
// .app bundles or shell scripts
if (file.endsWith('.sh')) {
  spawn('bash', [file], { cwd: workingDir });
}
```

## Download System

### Progress Tracking

```typescript
interface DownloadProgress {
  downloadedBytes: number;  // Bytes downloaded
  totalBytes: number;       // Total file size
  bytesPerSecond: number;   // Download speed
  timeRemaining: number;    // Estimated seconds remaining
  percent: number;          // 0-100
}

// Send progress to renderer
onDownloadProgress?.((progress: DownloadProgress) => {
  getMainWindow()?.webContents.send('download-progress', gameId, progress);
});
```

### Pause/Resume

```typescript
// Pause current download
const pausedState = pauseCurrentDownload(gameId);

// Check for paused state
const state = getPausedDownloadState(gameId);

// Resume from partial file
await resumeDownload(state, onDownloadProgress, onStatus);
```

## Backup System

Backups are stored with `.lbk_backup` suffix:

```typescript
import { backupFiles } from './backup';
import { restoreBackupNew, restoreBackupLegacy } from './backup';

// Create backup before installation
await backupFiles(gamePath, filesToBackup);
// Creates: file.ext.lbk_backup

// Restore from backup (new system with component tracking)
await restoreBackupNew(gamePath, installedFiles);

// Legacy restore (for old installations)
await restoreBackupLegacy(backupDir, gamePath);
```

## Installation Cache

Metadata stored in `~/.lbk-launcher/installations/{gameId}.json`:

```typescript
interface InstallationInfo {
  gameId: string;
  version: string;
  installedAt: string;        // ISO date
  gamePath: string;
  hasBackup?: boolean;
  protonPath?: string;        // For Linux Proton installations
  installedPlatform?: Platform; // steam, epic, gog, etc.
  components?: {
    text: InstallationComponent;  // Required when components present
    voice?: InstallationComponent;
    achievements?: InstallationComponent;
  };
}

interface InstallationComponent {
  installed: boolean;
  files: string[];           // Relative paths
  archiveHash?: string;      // For update detection
}
```

## Proton Integration

For Linux, Windows installers run via Proton:

```typescript
import { findProtons, runProton } from './proton';

// Find available Proton versions
const protons = findProtons();
// Returns: [{ name: 'Proton 8.0', path: '/path/to/proton' }, ...]

// Run installer with Proton
await runProton(
  protonPath,           // Path to Proton installation
  installerPath,        // Windows executable
  gamePath,             // Game installation directory
  onStatus              // Status callback
);
```

Proton prefix management:
- Uses existing Steam prefix if game is from Steam
- Creates prefix in `~/lbk-proton-prefixes/` for non-Steam games

## Archive Extraction

Supports multiple formats via 7z:

```typescript
await extractArchive(
  archivePath,
  destinationPath,
  {
    password?: string,        // For encrypted archives
    onProgress?: (percent) => void,
  }
);
```

Handled formats: `.zip`, `.7z`, `.rar`, `.tar.gz`, `.tar.bz2`

## Common Patterns

### Status Updates

```typescript
onStatus?.({ 
  status: 'downloading', 
  message: 'Завантаження файлів...' 
});

onStatus?.({ 
  status: 'extracting', 
  message: 'Розпакування архіву...' 
});

onStatus?.({ 
  status: 'installing', 
  message: 'Встановлення перекладу...' 
});
```

### Cleanup on Error

```typescript
try {
  await downloadAndExtract(...);
} catch (error) {
  // Clean up partial files
  await cleanupDownloadDir(downloadPath);
  throw error;
}
```
