# Main Process Instructions

This directory contains Electron main process code running in a Node.js environment.

## Environment

- **Runtime**: Node.js (full API access)
- **Accessible APIs**: `fs`, `child_process`, `path`, `os`, `electron` main APIs
- **Database**: better-sqlite3 (synchronous SQLite bindings)

## Directory Structure

```
src/main/
├── index.ts              # Entry point, app lifecycle, Sentry init
├── window.ts             # BrowserWindow creation and management
├── installer.ts          # Main installation orchestrator

├── tracking.ts           # Analytics event tracking

```

## Key Patterns

### IPC Handler Registration

All IPC handlers are grouped by feature in `ipc/` directory:

```typescript
// src/main/ipc/games.ts
export function setupGamesHandlers(): void {
  ipcMain.handle('fetch-games', async (_, params) => {
    // Handler implementation
  });
}

// src/main/index.ts - register handlers
setupGamesHandlers();
setupInstallerHandlers();
```

### Window Access

Use getter function, never cache window reference:

```typescript
import { getMainWindow } from './window';

// CORRECT - Always get fresh reference
getMainWindow()?.webContents.send('event', data);

// WRONG - Window might be destroyed
const win = getMainWindow();
// ... later
win.webContents.send('event', data); // May crash
```

### Error Handling

Use custom error classes for installer errors:

```typescript
import { ManualSelectionError, NetworkError } from './installer/errors';

if (!gamePath) {
  throw new ManualSelectionError('Game not found');
}

try {
  await download(url);
} catch (e) {
  throw new NetworkError('Download failed', { cause: e });
}
```

### Logging Convention

Use module prefix for easy filtering:

```typescript
console.log('[Module] Action description');
console.error('[Module] Error:', error);

// Examples:
console.log('[Installer] Starting installation for:', game.name);
console.log('[GameDetector] Found Steam at:', steamPath);
console.log('[Database] Running migration:', migrationName);
```

## Platform Considerations

### Always Check Platform

```typescript
import { isWindows, isLinux, isMacOS } from './utils/platform';

if (isWindows()) {
  // Windows-specific code
} else if (isLinux()) {
  // Linux-specific code (consider Proton)
} else if (isMacOS()) {
  // macOS-specific code
}
```

### File Paths

```typescript
import { app } from 'electron';
import path from 'path';

// User data directory
const userDataPath = app.getPath('userData');

// Resources directory (bundled assets)
const resourcesPath = app.isPackaged
  ? path.join(process.resourcesPath)
  : path.join(__dirname, '../../resources');
```

### Spawning Processes

```typescript
import { spawn } from 'child_process';

// Windows .bat files need shell: true
if (isWindows() && file.endsWith('.bat')) {
  spawn(file, args, { shell: true, cwd: workingDir });
}

// Linux with Proton
if (isLinux() && file.endsWith('.exe')) {
  await runProton(protonPath, file, gamePath);
}
```

## Performance Guidelines

1. **Use worker thread for heavy DB operations** - See `db/db-worker.ts`
2. **Don't block main thread** - Use async operations
3. **Cache expensive lookups** - Game detection, Steam library
4. **Stream large files** - Don't buffer entire downloads in memory

## Security Notes

1. **Never eval untrusted code**
2. **Sanitize file paths** - Prevent path traversal
3. **Validate IPC arguments** - Don't trust renderer input
4. **Use shell: true carefully** - Only for trusted commands
