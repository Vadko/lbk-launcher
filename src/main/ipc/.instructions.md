# IPC Handlers Instructions

This directory contains all IPC handler registrations for main process.

## Structure

```
src/main/ipc/
├── games.ts          # Game data, detection, search handlers
├── installer.ts      # Installation, uninstallation handlers  
├── tg-news.ts        # Telegram news feed handlers
└── window-controls.ts # Window minimize/maximize/close handlers
```

## Handler Naming Convention

Channel names should be:
- Lowercase with hyphens: `fetch-games`, `install-translation`
- Descriptive: `detect-game-platforms`, `check-installation`
- Consistent: `get-*` for queries, `set-*` for mutations

## Registration Pattern

```typescript
// games.ts
export function setupGamesHandlers(): void {
  // Query handler
  ipcMain.handle('fetch-games', async (_, params: GetGamesParams) => {
    const repo = GamesRepository.getInstance();
    return repo.getGames(params);
  });
}

// installer.ts
export function setupInstallerHandlers(): void {
  // Installation handler with optional custom path
  ipcMain.handle('install-translation', async (_, game, options, customGamePath?) => {
    return installTranslation(game, options, customGamePath, ...callbacks);
  });
}

// index.ts - called from main entry
setupGamesHandlers();
setupInstallerHandlers();
setupWindowControls();
```

## Handler Categories

### games.ts
- `fetch-games` - Get games with pagination/filtering
- `fetch-teams` - Get all translation teams
- `fetch-filter-counts` - Get counts for sidebar filters
- `detect-game-platforms` - Find where game is installed
- `find-games-by-steam-app-ids` - Match Steam library
- `sync-kurin-games` - Sync with Kurin mod manager

### installer.ts  
- `install-translation` - Install localization
- `uninstall-translation` - Remove localization
- `check-installation` - Get installation status
- `pause-download` / `resume-download` - Download control
- `get-conflicting-translation` - Check for conflicts

### window-controls.ts
- `window-minimize` - Minimize window
- `window-maximize` - Toggle maximize
- `window-close` - Close window
- `window-is-maximized` - Check maximize state

## Adding New Handler

1. Add handler in appropriate file:
```typescript
ipcMain.handle('new-handler', async (_, arg1: Type1) => {
  // Implementation
  return result;
});
```

2. Update `ElectronAPI` interface in `src/shared/types.ts`

3. Expose via contextBridge in `src/preload/index.ts`

## Error Handling

Return structured errors, don't throw:

```typescript
ipcMain.handle('operation', async () => {
  try {
    const result = await doOperation();
    return { success: true, data: result };
  } catch (error) {
    return { 
      success: false, 
      error: {
        message: error.message,
        code: error.code,
      }
    };
  }
});
```

## Event Emission (Main → Renderer)

Use `webContents.send` for one-way events:

```typescript
import { getMainWindow } from '../window';

// Emit progress updates
getMainWindow()?.webContents.send('download-progress', gameId, progress);

// Emit state changes
getMainWindow()?.webContents.send('game-updated', updatedGame);
```
