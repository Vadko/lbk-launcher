# Shared Module Instructions

This directory contains types and utilities shared between main and renderer processes.

## Module Structure

```
src/shared/
├── types.ts          # Core TypeScript type definitions
├── formatters.ts     # Formatting utilities (bytes, time)
├── search-utils.ts   # Search query helpers
└── api-config.ts     # API configuration
```

## Usage

Shared code can be imported from both processes:

```typescript
// From main process
import type { Game, InstallOptions } from '../shared/types';

// From renderer process
import type { Game } from '@/shared/types';
import { formatBytes } from '@/shared/formatters';
```

## Key Types

### Game

```typescript
interface Game {
  id: string;
  slug: string;
  name: string;
  team: string;
  description?: string;
  cover_url?: string;
  downloads: number;
  version?: string;
  install_paths?: InstallPath[];
  platforms?: string[];
  // ... additional fields
}
```

### InstallPath

```typescript
interface InstallPath {
  type: Platform;    // 'steam' | 'gog' | 'epic' | 'xbox' | 'rockstar'
  path: string;      // Relative path within platform library
}
```

### InstallOptions

```typescript
interface InstallOptions {
  platform: Platform | 'auto';
  installText: boolean;
  installVoice: boolean;
  installAchievements: boolean;
  createBackup: boolean;
  protonPath?: string;    // Linux only
}
```

### InstallationInfo

```typescript
interface InstallationInfo {
  gameId: string;
  version: string;
  installedAt: string;
  gamePath: string;
  hasBackup?: boolean;
  protonPath?: string;           // For Linux Proton installations
  installedPlatform?: Platform;  // steam, epic, gog, etc.
  components?: {
    text: InstallationComponent;  // Required when components present
    voice?: InstallationComponent;
    achievements?: InstallationComponent;
  };
}
```

### DownloadProgress

```typescript
interface DownloadProgress {
  downloadedBytes: number;  // Bytes downloaded
  totalBytes: number;       // Total file size
  bytesPerSecond: number;   // Download speed
  timeRemaining: number;    // Seconds remaining
  percent: number;          // 0-100
}
```

### ElectronAPI

The `ElectronAPI` interface defines all methods exposed to renderer:

```typescript
interface ElectronAPI {
  // Games
  fetchGames: (params: FetchGamesParams) => Promise<Game[]>;
  fetchGamesByIds: (ids: string[]) => Promise<Game[]>;
  
  // Installation
  installTranslation: (game: Game, options: InstallOptions) => Promise<InstallResult>;
  uninstallTranslation: (gameId: string) => Promise<void>;
  
  // Platform Detection
  detectGamePlatforms: (game: Game) => Promise<GamePath[]>;
  
  // Events (return unsubscribe function)
  onDownloadProgress: (callback: (id: string, progress: DownloadProgress) => void) => () => void;
  onInstallationStatus: (callback: (id: string, status: InstallationStatus) => void) => () => void;
  
  // ... many more methods
}
```

**Important**: When adding new IPC handlers, always update this interface!

## Formatters

```typescript
import { formatBytes, formatTime } from '@/shared/formatters';

formatBytes(1024);        // "1.00 KB"
formatBytes(1048576);     // "1.00 MB"
formatTime(125);          // "2:05"
formatTime(3600);         // "60:00"
```

## Type-Only Imports

When importing only types, use `import type`:

```typescript
// CORRECT - Type-only import
import type { Game, InstallOptions } from '@/shared/types';

// AVOID - Regular import for types
import { Game, InstallOptions } from '@/shared/types';
```

This ensures types are erased at compile time and don't create runtime dependencies.

## Platform Type

```typescript
type Platform = 'steam' | 'gog' | 'epic' | 'xbox' | 'rockstar';
```

Note: Platform type is derived from Supabase database enum. When adding new platforms, update both the database and this type.

## Guidelines

1. **Keep shared code minimal** - Only truly shared logic
2. **No process-specific imports** - No `electron`, `fs`, `react` imports here
3. **Pure functions** - Formatters should be side-effect free
4. **Type exports** - Prefer `export type` when possible
5. **Document complex types** - Add JSDoc comments for clarity
