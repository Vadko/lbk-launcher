# Preload Script Instructions

This file is the security bridge between main and renderer processes.

## File: `src/preload/index.ts`

## Purpose

The preload script:
1. Runs in renderer context but with Node.js access
2. Exposes selected main process APIs to renderer via `contextBridge`
3. Prevents renderer from accessing dangerous Node.js APIs directly

## Architecture

```
┌─────────────┐     contextBridge     ┌──────────────┐
│   RENDERER  │◄────────────────────►│   PRELOAD    │
│   (React)   │  window.electronAPI   │              │
│             │                       │              │
│  No Node.js │                       │  Node.js OK  │
│   access    │                       │  (sandboxed) │
└─────────────┘                       └──────┬───────┘
                                             │ ipcRenderer
                                             ▼
                                      ┌──────────────┐
                                      │    MAIN      │
                                      │   (Node.js)  │
                                      └──────────────┘
```

## Exposed APIs

All renderer-accessible methods go through `window.electronAPI`:

```typescript
contextBridge.exposeInMainWorld('electronAPI', {
  // Invoke pattern (request/response)
  fetchGames: (params) => ipcRenderer.invoke('fetch-games', params),
  installTranslation: (game, options, customGamePath?) => 
    ipcRenderer.invoke('install-translation', game, options, customGamePath),
  
  // Event listeners (return unsubscribe function)
  onDownloadProgress: (callback) => {
    const handler = (_, gameId, progress) => callback(gameId, progress);
    ipcRenderer.on('download-progress', handler);
    return () => ipcRenderer.removeListener('download-progress', handler);
  },
});
```

## Adding New API

### Step 1: Define IPC handler in main process

```typescript
// src/main/ipc/feature.ts
ipcMain.handle('new-feature', async (_, arg1, arg2) => {
  return result;
});
```

### Step 2: Add type to ElectronAPI interface

```typescript
// src/shared/types.ts
interface ElectronAPI {
  newFeature: (arg1: Type1, arg2: Type2) => Promise<ResultType>;
}
```

### Step 3: Expose in preload

```typescript
// src/preload/index.ts
newFeature: (arg1, arg2) => ipcRenderer.invoke('new-feature', arg1, arg2),
```

### Step 4: Use in renderer

```typescript
// src/renderer/components/Feature.tsx
const result = await window.electronAPI.newFeature(arg1, arg2);
```

## Event Patterns

### One-way events (Main → Renderer)

```typescript
// Preload
onGameUpdated: (callback) => {
  const handler = (_, game) => callback(game);
  ipcRenderer.on('game-updated', handler);
  return () => ipcRenderer.removeListener('game-updated', handler);
},

// Renderer
useEffect(() => {
  const unsubscribe = window.electronAPI.onGameUpdated((game) => {
    // Handle update
  });
  return () => unsubscribe();
}, []);
```

### Request/Response (Renderer → Main → Renderer)

```typescript
// Preload
installTranslation: (game, options) => 
  ipcRenderer.invoke('install-translation', game, options),

// Renderer
const result = await window.electronAPI.installTranslation(game, options);
```

## Window Controls

Separate exposure for window control functions:

```typescript
contextBridge.exposeInMainWorld('windowControls', {
  minimize: () => ipcRenderer.send('window-minimize'),
  maximize: () => ipcRenderer.send('window-maximize'),
  close: () => ipcRenderer.send('window-close'),
  isMaximized: () => ipcRenderer.invoke('window-is-maximized'),
});
```

## Security Guidelines

### DO
- Validate arguments before passing to ipcRenderer
- Use specific channel names (not generic)
- Clean up event listeners

### DON'T
- Expose `require` or `process` to renderer
- Pass unsanitized user input to shell commands
- Expose file system access without validation

```typescript
// DANGEROUS - Never do this
contextBridge.exposeInMainWorld('node', {
  require: require,           // ❌
  process: process,           // ❌
  fs: require('fs'),          // ❌
});

// SAFE - Specific, controlled APIs
contextBridge.exposeInMainWorld('electronAPI', {
  readGameConfig: (gameId) => ipcRenderer.invoke('read-game-config', gameId),  // ✅
});
```

## Type Safety

TypeScript declarations for window APIs:

```typescript
// src/window.d.ts
interface Window {
  electronAPI: ElectronAPI;
  windowControls: WindowControls;
}
```

This ensures renderer code has proper type checking for exposed APIs.
